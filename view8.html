주석표시

- 짧은 주석의 경우 //내용로 한줄로 표시한다.

- 긴 주석의 경우 /*내용*/으로 표시하는데, 이 때 연산자 * 등이 내용에 있을 경우 주석이 중간에 풀리게 될 수 있으니 사용을 지양하는게 좋다

식별자

- 변수/상수 선언시 사용되는 식별자에서는 $(달러표시)를 사용할 수 있지만, 현업에서 Jquery 문법과 혼동될 수 있으니 $ 표시는 최대한 사용을 지양하는게 좋다.

- 식별자로 한글을 이용할 수는 있으나, 아래의 문제가 있어 지양하는게 좋다.
① 용량문자 (한글은 최소 2바이트 이상, 영어는 1바이트)
② 인코딩 문제 발생 가능

연산자

- 연산자의 경우 =기호와 *, +, - 등 다른 연산자 기호와 함께 사용할 때가 많은데 무조건 함께 사용되는 연산자는 =의 좌측에 위치한다.

- 파이썬에서는 //으로 나누기 후 몫을 구해주는 연산자가 별도로 있지만 JS에는 따로 없어서 /으로 값을 구한 후 parseInt(값)으로 정수로 변환해주는 작업이 필요하다.

- 숫자+"문자열"의 경우 +연산자가 더하는 연산자이기도 하지만 문자열끼리 연결시켜주는 연결연산자이기도 하기때문에 숫자형을 자동으로 문자열로 변경해주게 된다.

- 나머지 다른 연산자의 경우 문자열이 포함된 연산의 경우 경우데 따라 아스키 코드로 변환해서 연산됩니다.

제어문

- if문을 사용할 때는 가능한 else도 함께 써서 예외처리를 해줄 수 있도록 해주는게 좋다.

- while문을 사용할 때는 반드시 반복문을 탈출 할 수 있도록 break등의 탈출점을 만들어줘야합니다.


REVIEW

- CSS 진도가 모두 나갔다. 추가로 필요한 부분은 따로 강의를 들으려고한다.

- JS에 대해 수업을 들으면서 그냥 쓰던 내용들에 대해 왜 그렇게 사용하는지 어떤게 더 올바른 방법인지에 대해 알 수 있었다.

- 특히 연산자에 대한 것은 따로 정리된걸로 공부한적이 없어서 이번기회에 연산자에 대해 공부할 수 있어서 굉장히 흥미로웠다.
